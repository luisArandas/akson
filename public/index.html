synopsis.
Akson is the name of a networked audio-visual system that allows large-scale playback and collaboration between artists. This environment was developed to fulfil the needs of a research in interactive networked musical systems and complex data visualizations done in the context of the Braga Media Arts group. Akson was initially conceived of as an exploration into what could be built by leveraging global internet infrastructure as well as musical and visual playback across multiple devices. This system was done thinking about its use in live performance and is able to interact with the devices of the public.

keywords.
Computer Music, New Media Art, Networked Art, Digital Interaction, Interfacing Art

  Interaction Methods in the Akson Network

    descenter
        This is the instrument default interaction model when the user connects. The descenter method lets the user stream the synthesizer through every device being able to control all the other atributes of the instrument just by himself. This method is based on The Improvisation Model, paradigm of the Jazz Combo (Winkler, 1998. p.25)

    streamed
        The streamed interaction state allows everyone to be a master. Every tiny change in the interface will stream to every user that also shares this state, and it is possible to create large-scale AV structures. This method is based on The Conductor Model, paradigm of the Symphony Orchestra (Winkler, 1998. p.23)

    allocate
        The allocate mode relies on the choice of the user on what to control. This method splits the instrument in four parts, where the other users are automatically allocated on what the current user didn't choose. The system is divided until cancelled. This method is based on The Chamber Music Model, paradigm of the String Quartet (Winkler, 1998. p.25)

    alone
        The alone mode allows the user to stop external collaboration. It ends and re-establishes the connections with other users, starting in the last method of interaction.
